# 04 — Tagging UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Enable end-to-end manual tagging: select transcript text → apply tag(s) → see span in right pane → add memo → play looped audio of that span. Tags update codebook counts. Multiple tags per span. Multiple overlapping spans per text.

**Architecture:** Backend gains `tagged_span` + `span_tag` + `memo` query modules and Tauri commands for span CRUD. Frontend adds selection-handling on the transcript viewer, a tag popover, span highlighting, right-pane span detail, an audio player. Audio uses Tauri's `convertFileSrc` to expose `media/*.mp3` files via the asset protocol.

**Tech Stack:** Plans 01–03 plus: native HTML5 `<audio>` element, `tauri-plugin-asset-protocol` for local file URLs.

**Spec reference:** §4 (tagged_span, span_tag, memo), §7.4 (manual tagging flow).

**Prerequisites:** Plans 01–03 merged.

## Task 1: `tagged_span` query module

**API:**

```rust
pub struct TaggedSpan {
    pub id: i64,
    pub interview_id: i64,
    pub segment_id: i64,
    pub start_offset: i32,
    pub end_offset: i32,
    pub text_snapshot: String,
    pub audio_start_sec: f64,
    pub audio_end_sec: f64,
    pub created_at: String,
}

pub struct NewSpan<'a> {
    pub interview_id: i64,
    pub segment_id: i64,
    pub start_offset: i32,
    pub end_offset: i32,
    pub text_snapshot: &'a str,
    pub audio_start_sec: f64,
    pub audio_end_sec: f64,
}

pub fn create(conn: &Connection, new: &NewSpan) -> AppResult<TaggedSpan>;
pub fn update_offsets(conn: &Connection, id: i64, start_offset: i32, end_offset: i32, text_snapshot: &str) -> AppResult<()>;
pub fn delete(conn: &Connection, id: i64) -> AppResult<()>;
pub fn get(conn: &Connection, id: i64) -> AppResult<TaggedSpan>;
pub fn list_for_interview(conn: &Connection, interview_id: i64) -> AppResult<Vec<TaggedSpan>>;
pub fn list_for_segment(conn: &Connection, segment_id: i64) -> AppResult<Vec<TaggedSpan>>;
pub fn list_for_tag(conn: &Connection, tag_id: i64) -> AppResult<Vec<TaggedSpan>>;
```

**Audio interpolation helper:**

```rust
pub fn interpolate_audio_range(
    segment_start: f64,
    segment_end: f64,
    segment_text_len: usize,
    char_start: i32,
    char_end: i32,
) -> (f64, f64);
```

Linear interpolation: `audio_start = segment_start + (segment_end - segment_start) * (char_start / segment_text_len)`. Clamp to `[segment_start, segment_end]`. 5 tests:
- Span covering full segment text → audio matches segment.
- Span at start → audio_start = segment_start.
- Span at end → audio_end = segment_end.
- Empty segment text (len 0) → both = segment_start.
- Span beyond text len → clamped to segment_end.

Module tests (6): create + read round-trip, multiple spans per segment, list_for_interview ordering by `(segment_id, start_offset)`, list_for_tag with join, update_offsets persistence, delete cascades span_tag rows.

Commit: `feat(db): tagged_span queries and audio interpolation`.

## Task 2: `span_tag` query module

```rust
pub enum SpanTagSource { Manual, AiSuggested, AiAccepted }

pub fn attach(conn: &Connection, span_id: i64, tag_id: i64, source: SpanTagSource) -> AppResult<()>;
pub fn detach(conn: &Connection, span_id: i64, tag_id: i64) -> AppResult<()>;

```

`attach` uses `INSERT OR IGNORE` — re-attaching the same (span, tag) is a no-op (returns Ok), not an error. The PK enforces uniqueness.

`set_source` is used by AI flows in Plan 05 to flip `ai_suggested` → `ai_accepted`.

Deletion: `tag` cannot be deleted if span_tag references it (FK RESTRICT from Plan 02). Already covered by Plan 02 tests.

5 tests.

## Task 4: Tauri commands — tagging

**File:** `src-tauri/src/commands/tagging.rs`

Commands:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SpanDTO {
    pub id: i64,
    pub interview_id: i64,
    pub segment_id: i64,
    pub start_offset: i32,
    pub end_offset: i32,
    pub text_snapshot: String,
    pub audio_start_sec: f64,
    pub audio_end_sec: f64,
    pub created_at: String,
    pub tags: Vec<TagOnSpan>,
    pub memo: Option<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TagOnSpan {
    pub tag_id: i64,
    pub source: String, // "manual" | "ai_suggested" | "ai_accepted"
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CreateSpanArgs {
    pub interview_id: i64,
    pub segment_id: i64,
    pub start_offset: i32,
    pub end_offset: i32,
    pub tag_ids: Vec<i64>,
}

#[tauri::command]
pub async fn span_create(app: tauri::AppHandle, args: CreateSpanArgs) -> AppResult<SpanDTO>;

#[tauri::command]
pub async fn span_update_tags(app: tauri::AppHandle, span_id: i64, tag_ids: Vec<i64>) -> AppResult<SpanDTO>;

#[tauri::command]
pub async fn span_update_offsets(
    app: tauri::AppHandle,
    span_id: i64,
    start_offset: i32,
    end_offset: i32,
) -> AppResult<SpanDTO>;

#[tauri::command]
pub async fn span_delete(app: tauri::AppHandle, span_id: i64) -> AppResult<()>;

#[tauri::command]
pub async fn span_get(app: tauri::AppHandle, span_id: i64) -> AppResult<SpanDTO>;

#[tauri::command]
pub async fn span_list_for_interview(app: tauri::AppHandle, interview_id: i64) -> AppResult<Vec<SpanDTO>>;

#[tauri::command]
pub async fn memo_upsert(app: tauri::AppHandle, span_id: i64, body: String) -> AppResult<()>;
```

Implementation notes:
- `span_create`: load the parent segment, validate `0 <= start_offset < end_offset <= segment.text.len()`. Compute interpolated audio range. Insert span, then attach each tag with `source = Manual`. Build & return SpanDTO with joined tags + (initially absent) memo.
- `span_update_tags`: compute diff between existing and new tag_ids; detach removed, attach added (manual source). Return full DTO.
- `span_update_offsets`: re-fetch segment, recompute text_snapshot and audio interpolation. Returns updated DTO.

Tests in `tagging_commands.rs`:
- create span attaches tags
- can't create span outside segment text bounds → `AppError::Invalid`
- update_tags is idempotent (same input → same output)
- delete cascades span_tag and memo
- 6 tests total

Register all commands in `lib.rs`.

Commit: `feat(commands): tagging CRUD with SpanDTO`.

## Task 6: Atoms

**File:** `src/state/tagging.ts`

```ts
import { atom } from "jotai";
import type { SpanDTO } from "../ipc/tagging";

export const spansForCurrentInterviewAtom = atom<SpanDTO[]>([]);
export const selectedSpanIdAtom = atom<number | null>(null);

export const selectedSpanAtom = atom((get) => {
  const id = get(selectedSpanIdAtom);
  if (id === null) return null;
  return get(spansForCurrentInterviewAtom).find((s) => s.id === id) ?? null;
});

export type ActiveSelection = {
  segmentId: number;
  startOffset: number;
  endOffset: number;
  text: string;
} | null;

export const activeTextSelectionAtom = atom<ActiveSelection>(null);
```

## Task 7: TranscriptViewer — selection handling + span highlighting

**File:** `src/views/Workspace/CenterPane/TranscriptViewer.tsx`

Behavior changes:
1. Wrap segment text in `<span data-segment-id={s.id} data-text="...">{s.text}</span>`.
2. Add a `mouseup` handler on the transcript container. On mouseup, get `window.getSelection()`:
   - If selection is non-collapsed and contained within a single segment span, compute `startOffset` and `endOffset` relative to the segment text (use `Range.startOffset` after walking up to the parent segment span).
   - If valid, set `activeTextSelectionAtom`.
   - Otherwise clear it.
3. Render existing spans as inline highlights:
   - For each `SpanDTO`, find its segment and overlay a colored background on chars `[startOffset, endOffset)` using `<mark>` elements with tag-color styling.
   - When multiple spans overlap, stack their marks (different background-colors with low alpha).
   - Clicking a `<mark>` sets `selectedSpanIdAtom`.
4. When `activeTextSelectionAtom` is non-null and selection is non-collapsed, render a floating `<TagPopover>` anchored near the selection.

This component grows substantially — keep it focused. Extract helpers:
- `useTextSelection(transcriptRef)` hook returning the `ActiveSelection`.
- `renderSegmentWithHighlights(segment, spans, codebook)` returns JSX for one segment with overlapping `<mark>` elements.

Aim for `TranscriptViewer.tsx` ≤ 200 LOC. If it grows, split `renderSegmentWithHighlights` into a `SegmentRow.tsx` component.

### Risks

- Text selection within a segment requires careful Range API handling. Be prepared to handle whitespace differences between rendered text and the source.
- Span highlight rendering with overlapping spans is non-trivial. May need a flatten-then-render approach: split each segment text into character ranges by intersection of all span boundaries, then emit a `<mark>` per range with composed styles.
- Asset protocol scope: starting with `**/*` for dev is fine; production scope should be the project directory only. Plan 08 polish item.
