# 07 — Export & Stats Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** All five export formats from spec §7.6 produce deterministic output verified by golden-file tests: CSV of tagged quotes, Markdown/HTML annotated transcript, REFI-QDA XML, stats report, codebook-only.

**Architecture:** One module per format under `src-tauri/src/export/`. Each module exposes a single function that takes a `ProjectExportData` snapshot and a writer. The snapshot is composed once by a shared helper. The frontend exposes an "Export" menu in the workspace with scope (current interview / project / filter by tag) and format selection.

**Tech Stack:** Plans 01–06 plus: `csv` crate, `quick-xml`, optionally `chrono` for timestamps.

**Spec reference:** §7.6.

**Prerequisites:** Plans 01–06 merged.

```rust

    pub project_name: String,
    pub interviews: Vec<InterviewExport>,
    pub clusters: Vec<cluster::Cluster>,
```

The composition runs joined queries to assemble everything in one pass. ~6 tests covering scopes (all, by interview, by tag).

## Task 5: REFI-QDA exporter

**File:** `src-tauri/src/export/refi_qda.rs`

REFI-QDA Project Exchange Format (XML). Reference: https://www.qdasoftware.org/products-project-exchange/

Minimal viable export covers:
- `<Project>` root with name, GUID.
- `<Users><User/></Users>` (single user).
- `<CodeBook><Codes>` recursive `<Code>` elements for clusters/categories/tags.
- `<Sources>` per interview with:
  - `<TextSource>` for transcript text.
  - `<AudioSource>` referencing the audio file (relative path).
- `<CodedRanges>` for each tagged span: source GUID, start/end character offsets in the text source, code GUIDs.

Use `quick-xml::Writer` for streaming XML emission. Generate stable UUIDs from internal IDs (deterministic for golden tests — e.g. `uuid::Uuid::new_v5(NAMESPACE, name)`).

Golden test against `tests/golden/export.refi`.

This is the most complex exporter. Budget 4 hours. Reference an existing QDA tool's export as a baseline if available.

Commit: `feat(export): REFI-QDA XML exporter`.

## Task 6: Stats exporter

**File:** `src-tauri/src/export/stats.rs`

Generates a stats report (CSV or Markdown) containing:

1. **Code frequency** by tag:
   ```
   cluster, category, tag, total_quotes, interviews_with_quote
   ```
2. **Frequency by interview**:
   ```
   interview, cluster, category, tag, count
   ```
3. **Frequency by speaker** (per interview):
   ```
   interview, speaker, tag, count
   ```
4. **Co-occurrence matrix** between tags (how often two tags appear on the same or adjacent spans):
   ```
   tag_a, tag_b, co_occurrence_count
   ```

```rust
pub fn write_stats_csv<W: Write>(data: &ProjectExportData, writer: &mut W) -> AppResult<()>;
pub fn write_stats_markdown<W: Write>(data: &ProjectExportData, writer: &mut W) -> AppResult<()>;
```

Co-occurrence definition: two tags co-occur when they're attached to the same span. (Adjacent-span co-occurrence is a future refinement.)

Golden tests.

Commit: `feat(export): stats exporter with frequencies and co-occurrence`.
