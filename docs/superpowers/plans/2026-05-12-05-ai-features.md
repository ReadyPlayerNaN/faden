# 05 — AI Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Three explicit, on-demand AI flows — codebook generation, bulk pre-tag, and find-more-like-this — each with a staging UI that lets the user review proposals before any DB mutation. A cost-confirmation modal precedes every AI call. Per-project prompt overrides editable in Settings.

**Architecture:** Shared `GeminiClient` (from Plan 03) is reused. New module `ai/` with one file per flow, each returning a typed `Proposal` enum. Each proposal is persisted to a staging table for review across sessions. Frontend gets a generic `StagingView` that handles accept/reject for each proposal type.

**Tech Stack:** Plans 01–04.

**Spec reference:** §5 (entire section), §8 (prompts), §9 (per-project prompt overrides).

**Prerequisites:** Plans 01–04 merged.

```sql
CREATE TABLE proposal (
    id INTEGER PRIMARY KEY,
    ai_run_id INTEGER NOT NULL REFERENCES ai_run(id) ON DELETE CASCADE,
    kind TEXT NOT NULL CHECK (kind IN ('codebook_gen','pretag','find_more')),
    payload_json TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending'
        CHECK (status IN ('pending','accepted','rejected')),
    created_at TEXT NOT NULL,
```

## Task 7: Codebook generation flow

**File:** `src-tauri/src/ai/codebook_gen.rs`

```rust
pub struct CodebookGenInput {
    pub interview_ids: Vec<i64>,
    pub include_existing_codebook: bool,
}

pub async fn run(
    app: &tauri::AppHandle,
    input: CodebookGenInput,
    api_key: &str,
    model: &str,
    prompt_override: Option<&str>,
) -> AppResult<i64>; // returns proposal id
```

Flow:
1. Load transcripts for each interview (concatenate text with segment markers).
2. Load existing codebook tree as compact text (if include_existing_codebook).
3. Render prompt (use override or default).
4. `ai_run::start`.
5. Call `GeminiClient::generate_content_no_file` (a variant that doesn't need a file upload — pure text prompt). Add this to gemini.rs.
6. Parse response as `CodebookProposal`.
7. `proposal::create` with serialized payload.
8. `ai_run::complete`.
9. Return proposal id.

On error: `ai_run::fail`, propagate error.

Tests with mocked Gemini response: 2 tests (success + invalid JSON retry).

Commit: `feat(ai): codebook generation flow`.

```rust
    model: &str,

```

Flow:
1. Load interview transcript (segments with ids visible).
2. Load codebook tree as compact text.
3. Render prompt.
4. `ai_run::start`, generate, parse `SpanSuggestions`.
5. Validate each suggestion:
   - `segment_id` exists in this interview.
   - `0 <= start_offset < end_offset <= segment.text.len()`.
   - Each `tag_name` matches a tag in the current codebook (case-insensitive). Drop suggestions with unknown tags (log to result_summary).
6. `proposal::create` with the filtered list.
7. `ai_run::complete`.

2 tests (success + filtering invalid suggestions).

- For each selected suggestion, create `tagged_span` + `span_tag` rows with `source = AiAccepted`.

## Task 15: Smoke verification

Manual:
1. Create project, add 1 interview, transcribe.
2. Click "Generate codebook" → CostPreview → Send → wait for staging notification.
3. Open staging → review proposed clusters/categories/tags → accept some, reject some → see codebook update.

5. Select a tag, click "Find more like this" → review → accept.
6. Open Settings → Prompts → tweak codebook_gen prompt → Save → run codebook gen again → confirm new prompt was used (check `ai_run.prompt` in DB).
